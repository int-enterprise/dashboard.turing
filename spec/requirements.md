# 프로젝트 요구사항

## 개요

사내 직원 전용 엔터프라이즈 대시보드. 과거 수행한 프로젝트에 도입된 AI Agent의 퍼포먼스를 모니터링한다. 고객사에게 노출되는 실사업 서비스이므로 디자인 퀄리티가 중요하다.

---

## 기술 스택

- **Framework:** Next.js (App Router)
- **Package Manager:** pnpm
- **Styling:** Tailwind CSS (컬러는 `spec/color-palette.md` 준수)
- **Language:** TypeScript
- **DB:** Vercel Postgres
- **ORM:** Prisma
- **Deployment:** Vercel (only)
- **Language (UI):** 한국어

> 외부 서비스는 일절 사용하지 않고, Vercel 내부에서 모든 인프라를 관리한다.

---

## 인증 / 계정 관리

### 로그인
- 사내 직원 전용
- 공개 회원가입 불가
- 방식: 아이디 + 비밀번호 (자체 구현)
- SSO / 2FA 사용 안 함
- 세션 유지 기간: **7일**

### 계정 모델 (AWS IAM 구조 차용)
- **Root 계정**: 1개
  - 일반 권한 + **다른 계정 관리 권한**
- **IAM 계정**: N개
  - Root 외 나머지 권한은 모두 동일
  - 프로젝트 접근 권한 차등 없음 (모든 IAM 계정이 모든 프로젝트 접근)
- 중간 관리자 롤 없음

### 계정 관리 기능 (Root 전용)
- IAM 계정 생성 / 수정 / 삭제
- Next.js 백엔드(API Routes 또는 Server Actions) + DB로 구현
- Root 계정은 **최초 DB seed**로 심는다

---

## 프로젝트

- 로그인 성공 후 프로젝트 선택 화면 노출
- 프로젝트는 상시 변경 가능
- 프로젝트 추가는 **수동 개발**로 처리 (신규 프로젝트가 생기면 코드·DB에 직접 추가)
- 각 프로젝트에는 AI Agent가 도입되어 있다

---

## 모니터링 대시보드

### 데이터 소스
- AI Agent 퍼포먼스 지표는 **외부 API**에서 가져옴 (우리 API 아님)
- 실시간 / 비실시간 혼재 (프로젝트마다 다름)
- 프로젝트마다 지표 종류/스키마가 **완전히 다름**
- ※ 외부 API 세부사항(endpoint, 인증, 주기 등)은 추후 결정

### UI / UX
- **Grafana 스타일 모방** (패널 그리드 기반 대시보드)
- 테마: **라이트 모드 기본**, 다크 모드 전환 가능
- 디자인 퀄리티 최우선 — 고객사 노출 가능 수준의 완성도
- 컬러는 `spec/color-palette.md` 팔레트 및 60/30/10 비율 엄수

### 패널 편집(드래그/리사이즈/추가 등)
- 추후 결정 (MVP 스코프 논의 필요)

---

## 미정 / 추후 결정 항목

- 외부 지표 API 상세 (endpoint, 인증, 호출 주기, 캐싱 전략)
- 프로젝트별 지표 스키마 정의 방식
- Grafana 수준 패널 편집 기능의 MVP 포함 여부
- 알림(Alert) 기능 도입 여부

---

## 디자인 원칙 (영구 준수)

1. `spec/color-palette.md` 색상만 사용
2. 사용 비율: Primary·Base 60% / Secondary 30% / Accent 10%
3. 허접한 인상 금지 — 엔터프라이즈 퀄리티 유지
